Postup pro testování

1. V "Test" User Define Variables provést úpravy nastavení
	=> upravit počet Uživatelských účtů, které budou využity 	(UserCount)
	=> nastavit index csv souboru 1 - 5 				(UserCsvFileIndex)
		UserCsvFileIndex změní nahrávaný csv soubor s emaily.
		Toto lze využití při pseudosychroním decentralizovaném testování.

2. Je nutné provést spuštění ThreadGroup Firebase API - CreateUser
	Tento krok vytvoří testovací uživatele ve Firebase.

3. Upravení potřebného ThreadGroup konkrétního testu
	=> nastavení "Ramp-up" period
	
	=> kromě OnBoardingu lze také vytvořit cyklické volání a zadat počet cyklů v "Loop Count"
		=> při využití LoopCount nebo nekonečného LoopCount je vhodné Využít Thread lifetime nastavovaný v sekundách
		=> OnBoarding vyváří v DB uživatele, při druhém volání vrací "Email exist"

	=> kromě mazání uživatšelů, kde je limtováno na 8 "delete" za sekundu

4. Po provedení zatěžovacích testů je zapotřebí provést smazání uživatelů
	=> Loono API Delete users
	
	=> Případně lze provést kontrolu prostřednictvím Firebase API Delete users
		Pokud Loono API Delete users dopadne kompletně v pořádku, Firebase API Delete users budou mít 100% Error rate.
		Při zcela správně provedeném Loono API Delete users, se odstraňuje i záznam o účtu z Firebase.

U Firebase je nutné provést distribuci vytváření uživatelů, a také dočasně navýšit na 1000 requestů (1000 je limit navýšení)

... https://firebase.google.com/docs/auth/limits?authuser=0&hl=en#api_limits

Account creation and deletion limits

	New account creation	100 accounts/IP address/hour
	Account deletion	10 accounts/second

	Note: You can schedule a temporary increase to the account creation limit in the Firebase console

Lze navýšit limit na 1000 účtů za hodinu.
https://console.firebase.google.com/u/0/project/loonoapp-dev/authentication/settings

Mazání navýšit nelze. Mazání je tedy uměle zpomaleno v rámci JMeter na bezpečných 8x "delete" za sekundu.
Zpomalení je řešeno dynamicky, podle počtu testovacích uživatelů (UserCount).
